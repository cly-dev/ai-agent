## 为什么

项目已完成初始化并定义了 Prisma 数据模型，但当前缺少一套统一、可维护的代码生成能力，无法从 `prisma/schema.prisma` 快速得到对应的 TypeScript 类型、实体类与 Nest 模块骨架。随着模型增多，手工维护会带来重复劳动和类型不一致风险。

## 变更内容

- 新增基于 `prisma/schema.prisma` 的生成流程，用于自动生成模型类型定义、实体类文件与模块基础结构。
- 约定生成产物目录、命名规范和覆盖策略，保证重复执行可预期（幂等）。
- 增加生成命令与使用文档，明确开发者在模型变更后的标准操作步骤。
- 定义生成时的关系字段处理规则（例如一对多、多对多中间表）与可选字段映射。

## 功能 (Capabilities)

### 新增功能
- `prisma-schema-codegen`: 基于 Prisma schema 自动生成类型、实体类和对应模块骨架，并提供统一命令触发。

## 影响

- 受影响代码：新增代码生成脚本、生成模板、输出目录结构与文档。
- 受影响 API：无直接外部接口变更，主要是项目内部开发流程增强。
- 受影响依赖：可能增加用于 Prisma schema 解析或模板渲染的开发依赖。
- 受影响系统：Nest 应用的模块组织方式与数据库模型到业务代码的映射链路。

## 新增需求

### 需求:基于 Prisma Schema 生成类型
系统必须能够从 `prisma/schema.prisma` 自动生成与模型一致的 TypeScript 类型定义，且类型结果必须正确反映字段标量类型、可空性和默认可选属性。

#### 场景:全量生成类型成功
- **当** 开发者执行代码生成命令并指定 schema 路径
- **那么** 系统在约定目录输出全部模型对应的类型文件，且每个模型字段类型与 schema 一致

#### 场景:模型字段变更后类型更新
- **当** 开发者修改 schema 中某模型字段类型或可空性并重新执行生成
- **那么** 已生成类型文件必须同步更新，不得保留过期字段定义

### 需求:基于 Prisma Schema 生成实体类
系统必须能够为每个 Prisma 模型生成对应实体类，实体类字段必须与 schema 模型字段对齐，并支持关系字段的可配置输出策略。

#### 场景:生成实体类包含标量字段
- **当** 生成流程处理任一模型
- **那么** 输出实体类必须包含该模型全部标量字段，且命名与类型映射符合约定

#### 场景:生成实体类处理关系字段
- **当** 生成流程处理包含关系字段的模型
- **那么** 系统必须按配置生成关系引用字段（如外键或简化关系对象），并避免产生不可解析的循环导入

### 需求:基于 Prisma Schema 生成 Nest 模块骨架
系统必须为每个指定模型生成对应的 Nest 模块骨架，至少包含模块定义，并允许按模板扩展 service/controller 文件。

#### 场景:生成基础模块文件
- **当** 开发者执行模块生成并包含某模型
- **那么** 系统必须生成该模型对应的模块文件，并符合项目命名规则与目录约定

#### 场景:重复执行保持幂等
- **当** 开发者在未修改 schema 的情况下重复执行生成
- **那么** 生成结果必须稳定且不产生无意义文件差异

### 需求:代码生成必须可控且可审查
系统必须提供生成预览与覆盖控制能力，以便开发者在写入前确认变更范围并避免误覆盖手工代码。

#### 场景:dry-run 预览
- **当** 开发者以 dry-run 模式执行生成
- **那么** 系统只输出拟变更清单与差异摘要，不写入文件

#### 场景:覆盖策略保护手写代码
- **当** 生成目标目录存在手写文件且未显式开启覆盖
- **那么** 系统必须禁止覆盖并给出明确提示

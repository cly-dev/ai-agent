## ADDED Requirements

### 需求:业务用户必须基于身份分类控制可用 Tool 子集
系统必须依据业务用户的 `UserType` 与 `UserRole` 判定可调用 Tool 子集，并禁止未授权 Tool 调用。

#### 场景:用户调用已授权 Tool
- **当** 业务用户调用其所属 `UserType/UserRole` 已授权的 Tool
- **那么** 系统必须允许调用并执行

#### 场景:用户调用未授权 Tool
- **当** 业务用户调用未授权的 Tool
- **那么** 系统必须拒绝调用并返回权限不足

### 需求:用户权限必须与后台系统权限隔离
系统必须将业务用户 Tool 使用权限与后台管理员系统管理权限隔离，并禁止业务用户通过业务角色获得后台管理能力。

#### 场景:业务用户访问后台管理接口
- **当** 业务用户请求 Tool/Skill/Integration 管理接口
- **那么** 系统必须拒绝并提示无系统管理权限

### 需求:权限模型必须可扩展到未来 RBAC
系统必须在当前简单模型中保留可扩展接口，以支持未来升级为更细粒度 RBAC，而不改变 `User` 与 `AdminUser` 双主体边界。

#### 场景:新增业务角色映射
- **当** 系统新增一个 `UserRole` 并配置 Tool 授权映射
- **那么** 不需要重构用户主体模型即可生效

## ADDED Requirements

### 需求:业务用户与后台管理员必须使用独立主体模型
系统必须将业务用户与后台管理员建模为两个独立主体（`User` 与 `AdminUser`），并禁止后台管理权限字段直接落在业务用户主体上。

#### 场景:创建业务用户
- **当** 系统创建一个 C 端或 B 端业务用户
- **那么** 该用户数据仅写入 `User` 模型且不包含 `AdminRole`

#### 场景:创建后台管理员
- **当** 系统创建一个后台管理员账户
- **那么** 该账户数据仅写入 `AdminUser` 模型且必须包含 `AdminRole`

### 需求:用户身份分类必须显式表达
系统必须在 `User` 模型中显式记录业务身份分类（如 C 端外部用户、B 端内部员工），并禁止用后台管理员角色替代该分类。

#### 场景:查询业务用户身份
- **当** 系统读取业务用户基础信息
- **那么** 返回结果必须包含 `UserType` 或等价身份分类字段

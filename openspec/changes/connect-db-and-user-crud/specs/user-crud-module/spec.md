## 新增需求

### 需求:系统必须提供用户创建能力
系统必须提供创建用户接口，并将合法输入写入数据库后返回新建用户信息。

#### 场景:创建用户成功
- **当** 客户端提交合法的 `username`
- **那么** 系统必须创建用户并返回包含 `id`、`username`、`createdAt` 的结果

#### 场景:创建参数非法
- **当** 客户端提交空用户名或不符合校验规则的数据
- **那么** 系统必须返回参数错误并禁止写入数据库

### 需求:系统必须提供用户查询能力
系统必须提供用户列表查询和单用户详情查询能力。

#### 场景:查询用户列表
- **当** 客户端调用用户列表接口
- **那么** 系统必须返回用户集合

#### 场景:查询不存在用户
- **当** 客户端使用不存在的用户 ID 查询详情
- **那么** 系统必须返回资源不存在错误

### 需求:系统必须提供用户更新能力
系统必须允许更新用户可编辑字段，并返回更新后的用户数据。

#### 场景:更新用户成功
- **当** 客户端提交存在用户的合法更新字段
- **那么** 系统必须持久化变更并返回更新后的用户信息

#### 场景:更新不存在用户
- **当** 客户端更新不存在的用户 ID
- **那么** 系统必须返回资源不存在错误

### 需求:系统必须提供用户删除能力
系统必须支持删除指定用户，并对删除不存在用户给出明确响应。

#### 场景:删除用户成功
- **当** 客户端删除存在的用户 ID
- **那么** 系统必须删除该用户并返回成功结果

#### 场景:删除不存在用户
- **当** 客户端删除不存在的用户 ID
- **那么** 系统必须返回资源不存在错误

## 新增需求

### 需求:应用必须可建立数据库连接
系统必须在应用启动后可创建并复用数据库连接能力，以支持业务模块访问 PostgreSQL。

#### 场景:启动时连接可用
- **当** 应用启动并加载数据库模块
- **那么** 系统必须能够初始化 Prisma 客户端并建立可用连接

#### 场景:模块中可注入数据库服务
- **当** 业务模块声明依赖数据库服务
- **那么** 系统必须提供可注入实例并可执行模型查询

### 需求:数据库连接失败必须可观测
系统必须在数据库连接失败时提供明确错误反馈，禁止静默失败。

#### 场景:连接参数错误
- **当** `DATABASE_URL` 无效或数据库不可达
- **那么** 系统必须输出明确错误信息并阻止服务以错误状态运行
